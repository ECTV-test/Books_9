# Books_8 — Web Book Reader (Read + Listen)

Це статичний web‑додаток для читання та прослуховування коротких книжок з перекладом рядок‑в‑рядок.
Працює як звичайний сайт (GitHub Pages / будь‑який статичний хостинг), без бекенду.

## Що є в додатку

### 1) Бібліотека
- Каталог книг береться з `books/index.json` (один запит → одразу весь список).
- Картка книги показує: обкладинку, назву (по UI‑мові), автора, жанр/серію (`series`, напр. `NEW`), прогрес.

### 2) Три “мовні” поняття
- **UI language** (мова інтерфейсу) — впливає на:
  - назву книги (`title_<uiLang>`),
  - опис (`desc.<uiLang>.txt`),
  - переклад UI‑рядків.
- **Book language** (мова оригіналу книги) — текст, який читаємо.
- **Translation language** (мова перекладу) — другий стовпчик/рядок перекладу.

> Файли перекладу завжди шукаються за правилом: `book.<lang>.txt`.

### 3) Read / Listen
- **Read**: показує текст (і переклад, якщо вибрано), підтримує зміни розміру шрифту.
- **Listen**: режим прослуховування. Розмір шрифту та міжряддя синхронізовані з Read через CSS‑змінні.

### 4) Прогрес
- Прогрес зберігається локально у браузері (`localStorage`).
- Прогрес прив’язаний до:
  - книги,
  - пари мов (Book Lang + Translation Lang),
  - і є спільним для Read/Listen для цієї пари.
- У блоці «Продовжити читання» показується останній активний “пакет” (режим/мови) та відсоток.

### 5) Закладки
- Можна додавати закладку на слово/рядок.
- У «Bookmarks» є перехід назад у книгу та відкриття на потрібній позиції.

### 6) Теми (Light / Night)
- Перемикач теми змінює `data-theme` (використовується тема `night`).
- Кольори повинні братися через CSS‑змінні (`--bg`, `--card`, `--text`, `--muted`, …).
- Якщо якийсь елемент “не читається” в night — це зазвичай через hardcoded `color:#111` / `background:#fff`. Такі місця потрібно заміняти на змінні або додати night‑override.

## Структура репозиторію
- `index.html` — точка входу
- `css/styles.css` — стилі + тема
- `js/app.js` — логіка UI, завантаження книг, прогрес/закладки
- `books/` — книги + `books/index.json` (каталог)
- `tools/build_index.py` — генератор `books/index.json` з `books/*/book.json`

## Запуск локально
Потрібен будь‑який статичний сервер. Наприклад:

```bash
python3 -m http.server 8080
```

Відкрий у браузері:
`http://localhost:8080`

## Деплой на GitHub Pages
1. Залий репозиторій.
2. Увімкни Pages (branch `main`, folder `/`).
3. Переконайся, що папка `books/` лежить поруч з `index.html` в опублікованій збірці.

## Типові проблеми
### «Книга відкривається, але переклад порожній»
- Немає файла `books/<id>/book.<lang>.txt` → буде fallback на оригінал.
- Перевір, що назва файлу саме `book.xx.txt`, де `xx` — код мови зі списку в UI.

### «Рядки не співпадають між оригіналом та перекладом»
- У всіх `book.<lang>.txt` повинна бути **та сама кількість рядків**, що й у `book.txt`.
